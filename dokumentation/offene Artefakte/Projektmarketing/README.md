# Verzeichnisinformationen

Dieses Verzeichnis umfasst die [Projektmarketing-Vision als Markdown-Vorlage](Projektmarketing-Vision.md). Sollten Sie Bilder einbinden, dann sind diese im gleichen Verzeichnis abzulegen (siehe Beispiel).

**Diese Readme sollten Sie zur Abgabe von M1 überarbeiten. Die Aufgabenstellung und Erläuterungen können gelöscht werden.**

## Aufgabenstellung

Sie machen eine Zeitreise an das Ende ihres erfolreichen Projektes. Dazu erstellen Sie eine Projektmarketing-Vision *als wäre ihr Projekt schon erfolgreich verlaufen und die Ergebnisse würden veröffentlicht.*

## Erläuterung

Tatsächlich werden Sie am Ende ihres Projektes im Zuge der letzten Projektphasen eine solche Projektbeschreibung erstellen, die dann auf der WSF Webseite veröffentlicht wird. Dies haben schon die Jahrgänge vor Ihnen gemacht. Sie können [Beispiele aus den vergangenen Jahren auf dieser Webseite](https://www.th-wildau.de/wsf) einsehen. Für ihre finale Abgabe können Sie also hier vorarbeiten.

In dieser Abgabe am Anfang des Projektes machen Sie eine Zeitreise an das Ende ihres Projektes und schauen mit Hilfe dieser Webseite zurück. Was werden Sie erreicht haben? Was wird ihre Herangehensweise und Lösung (wahrscheinlich) gewesen sein?

Sie beschreiben dies auf der Projektmarketing-Vision-Webseite so, als wäre das Projekt bereits gelaufen und dies nun ihre letzte Abgabe. Dieses Werkzeug soll ihnen helfen eine positive Zukunftsvision zu entwickeln. Es ist auch ein Abgabeartefakt am Ende des Projektes, daher lohnt es sich, jetzt zu investieren, Feedback zu erhalten und am Ende nur noch ein Update der Webseite zu machen.

## Grundstruktur

Die Grundstruktur der Webseiten ist dabei immer gleich.

- Titel + kurze Zusammenfassung
- Kontext des Auftraggebers (ggf. Logo)
- Herausforderungen für das Projekt
- Herangehensweise
- Lösung
- Fazit
